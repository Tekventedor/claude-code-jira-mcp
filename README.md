# claude-code-jira-mcp

Motion-graphics promo for the FlowHunt blog **"How to Use Claude Code with the Jira MCP"**.

~69.3 seconds at 30 fps. 9 scenes. Built with [rendervid](https://github.com/qualityunit/rendervid).

## Brand assets

The real Claude AI mark + Atlassian two-chevron mark are inlined as base64 data URIs via `assets.mjs`. Source files are in `./assets/`. Regenerate by re-running the encode block in `assets.mjs` (or `base64 -i assets/<file> | tr -d '\n'`).

See `spec.md` for the frame-level beat sheet.

## Edit loop

1. Edit `build.mjs` (the source of truth — never edit `template.json` directly).
2. `node build.mjs` to regenerate `template.json`.
3. Click **Load** again in the playground at `http://localhost:5180/` and pick `template.json`.

## Render

```bash
node render.mjs
```

Outputs to `output/claude-code-jira-mcp.mp4`. The dev playground does not render; only this script does.

## Project layout

- `build.mjs` — inline scene components + template assembly
- `render.mjs` — Node renderer entrypoint
- `template.json` — generated by `build.mjs`, what the playground loads
- `source-blog.md` — the blog the video promotes (copy of `how-to-use-claude-code-with-the-jira-mcp.md`)
- `spec.md` — scene beat sheet
- `voiceover.md` — per-scene VO script the user records against (45 s, ~190 WPM)
- `how-to-use-claude-code-with-the-jira-mcp.md` — the blog (publish target)
- `demo-walkthrough.md` — script the user followed to capture the demo data referenced in this video
- `claude-code-jira-mcp-hero.svg`, `...-snapshot-vs-rendered.svg`, `...-flow-diagram.svg` — blog visuals
- `screenshots/` — raw captures + Kapture recording from the live demo (reference only, gitignored)
